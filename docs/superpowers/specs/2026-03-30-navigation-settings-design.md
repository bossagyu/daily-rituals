# ナビゲーション改善 & 設定ページ追加

## 概要

ボトムナビゲーションからログアウトボタンを除去し、新設する設定ページに移動する。ヘッダーからもユーザー名・ログアウトを削除してアプリ名のみに簡素化する。設定ページにはログアウトを配置する。

## 動機

- ボトムナビにログアウトがあると誤タップのリスクが高い
- ヘッダーにもログアウトがあり冗長
- ログアウトは頻繁に使う操作ではないため、設定ページに移動するのが適切

## 変更内容

### 1. ヘッダー（AppHeader）

**Before:** アプリ名 + ユーザー名 + ログアウトボタン
**After:** アプリ名のみ（中央寄せ）

### 2. ボトムナビゲーション（モバイル）

**Before:** Today | カレンダー | 習慣一覧 | ログアウト（4タブ）
**After:** Today | カレンダー | 習慣一覧 | 設定（4タブ、⚙️アイコン）

### 3. サイドナビゲーション（デスクトップ）

**Before:** Today / カレンダー / 習慣一覧 + 下部にログアウトボタン
**After:** Today / カレンダー / 習慣一覧 / 設定（ログアウトボタン削除）

### 4. 設定ページ（新規）

- ルート: `/settings`
- 最小構成のリスト形式

**項目:**
| 項目 | UI | 動作 |
|------|-----|------|
| ログアウト | ラベル「ログアウト」のみ（通常色、サブテキストなし） | タップで確認ダイアログ → ログアウト実行 |

ページ下部にバージョン情報を表示。

## デザイン方針

- ログアウトは特別な色を使わない（赤色にしない）。誤操作防止は確認ダイアログで対応
- 最小構成（YAGNI）。将来の項目追加はリストに行を足すだけ
- デスクトップ・モバイルで統一された挙動
